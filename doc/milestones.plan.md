## 二、M2 — 短期任务迭代 (当前)

### 目标

通过低粒度、短期可执行的任务计划，持续改进项目质量、安全性和用户体验。

### 计划管理方式

- 所有待实施任务存放于 `plan/` 目录，按编号排序
- 每个任务计划预估 0.5-2 天，包含具体检查清单
- 开发流程: 选取计划 → 实施 → 同步 doc/ → 验证 → 提交推送
- 详见 `CLAUDE.md` 中的 Development Workflow 章节
