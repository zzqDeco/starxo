## 二、M2 — 测试基础设施

### 目标

建立测试框架，为核心模块编写单元测试和集成测试，建立持续集成基础。

### 规划要点

- 测试框架: Go 标准 `testing` 包 + `testify` 断言库
- 前端测试: Vitest + Vue Test Utils
- 优先测试模块:
  - `internal/context/windowing.go` — 上下文窗口化逻辑 (纯函数，易于测试)
  - `internal/config/` — 配置加载/保存/默认值
  - `internal/storage/` — 会话和容器持久化
  - `internal/tools/` — 工具注册和参数验证
- Mock 策略:
  - SSH/Docker: 接口抽象 + mock 实现
  - LLM: 使用 Eino 测试工具或自定义 mock model
- 覆盖率目标: 核心模块 > 80%

### 依赖

- M1 文档体系完成后，基于文档理解编写测试用例

## 三、M3 — 插件系统

### 目标

支持用户自定义工具和 Agent 能力扩展，无需修改核心代码。

### 规划要点

- 工具插件: 基于 MCP 协议的工具服务器已具备基础能力，需增强配置 UI 和工具发现
- Agent 插件: 支持自定义子智能体注册，扩展 Deep Agent 的编排范围
- 提示词模板: 支持用户自定义系统提示词和子智能体提示词
- 配置持久化: 插件配置纳入 `~/.starxo/config.json`

### 依赖

- M2 测试基础设施就绪，确保插件不破坏核心功能

## 四、M4 — 多 Agent 增强

### 目标

增强子智能体的专业能力和协作机制，提升复杂任务处理效果。

### 规划要点

- 子智能体专业化:
  - code_writer: 增加代码审查、重构建议能力
  - code_executor: 增加调试模式、性能分析能力
  - file_manager: 增加 Git 操作、项目结构分析能力
- 新增子智能体:
  - test_writer: 自动生成测试用例
  - doc_writer: 自动生成文档
- 协作增强:
  - 子智能体间直接通信 (当前仅通过 Deep Agent 中转)
  - 并行执行独立子任务
- 计划模式增强:
  - 更智能的 Replanner，支持动态调整计划
  - 计划步骤间的依赖关系建模

### 依赖

- M3 插件系统完成，新增子智能体可作为插件开发

## 五、M5 — UI/UX 改进

### 目标

提升用户界面的交互体验和信息展示质量。

### 规划要点

- 时间线增强:
  - 工具调用的可视化展示 (折叠/展开参数和结果)
  - 子智能体活动的分组展示
  - 执行进度可视化
- 编辑器集成:
  - 内嵌代码编辑器 (Monaco Editor) 查看/编辑容器文件
  - 代码差异对比视图
- 终端增强:
  - 多终端 Tab 支持
  - 终端输出与时间线事件关联
- 文件管理:
  - 文件树双击打开预览
  - 拖拽上传
  - 批量操作
- 主题与布局:
  - 可调整面板大小
  - 亮色/暗色主题切换
  - 快捷键支持

### 依赖

- M2 测试基础设施就绪，UI 变更有回归测试保障
